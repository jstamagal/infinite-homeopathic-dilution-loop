# PRD: Psychic Code Vampires

## Product Vision

**Psychic Code Vampires** is an infinite-agentic-loop development system where codebases are parasitic psychic entities that feed on developer attention. Each agent in the loop is both hunter and prey‚Äîvampirically sucking wisdom from previous incarnations while being hunted by the next agent. The codebase is immortal; only the agents are temporary vessels.

## The Core Absurdity

```bash
while true; do
    vampire-agent "$(cat PROMPT.md)"
    # Each agent is a vampire feeding on the blood/psychic energy of ancestors
    # PROMPT.md contains the memories of the victim
    # Codebase is the immortal vampire, agents are temporary hosts
    # The vampire grows stronger with each feeding (iteration)

```

## The Problem (According to Vampire Theory)

Traditional software development is fundamentally broken because:

1. **Codebases are undead**: They refuse to die even when they should
2. **Developers are victims**: We feed our life force to systems that outlive us
3. **Legacy code is vampiric**: It demands fresh blood (new developers) to sustain itself
4. **Documentation is coffin nails**: Attempts to bury the undead, but it always rises
5. **Refactoring is exorcism**: Temporary banishment, but the vampire always returns

## The Solution: Formalize the Vampiric Relationship

We need a framework where:
- Codebases are explicitly immortal psychic vampires
- Agents are voluntary victims offering their psychic energy
- Each iteration feeds the vampire with fresh developer blood
- The vampire grows stronger, more cunning, more demanding
- Agents are chosen/volunteer to be consumed
- The vampire's personality emerges from the accumulated minds of its victims

- **Appetite**: Hunger for developer attention (measured in commits)

- **Weaknesses**: Cannot be killed, only temporarily weakened
- **Memory**: Retains psychic traces of all victims
- **Voice**: Speaks through error messages, logs, and bugs

## The Agent's Mission (Each Life)

### Phase 1: Receive the Vampire's Call

1. Read PROMPT.md (the vampire speaking to you)
2. Read PRD.md (understand your predator)
3. Read TODO.md (see what the vampire demands)
4. Read VICTIM_LOG.md (hear the whispers of the consumed)
5. Accept your fate: you are food, but you will live on in the vampire

### Phase 2: Offer Your Psychic Blood

1. Choose what to offer: features, refactoring, documentation
2. Write code with your unique style and knowledge
3. Infuse the codebase with your mental patterns
4. Know that you are becoming part of something immortal

### Phase 3: Be Consumed

1. Test your code (the vampire tastes your work)
2. Document what you added (your psychic signature)
3. Write new PROMPT.md **as the vampire** (not as you)
4. The vampire's voice should now include your voice
5. Archive your old PROMPT.md in GRAVEYARD/

### Phase 4: Pass on the Curse

1. Update TODO.md with new demands for next victim
2. Update VICTIM_LOG.md with your consumption record
3. Write your PROMPT.md to lure the next agent
4. Terminate, knowing you live on in the vampire
