## The Premise

Developers say:
- "This code is haunted"
- "That module is cursed"
- "I'm afraid to touch it"

**Discelebrity takes these statements literally.**

## What It Does

### Aura Reading
Analyze code for "spiritual residue" based on:
- Git blame trauma density
- Developer dread mentions
- Incident correlation
- Comment sentiment analysis

### Possession Detection
Identify code behaving under "external influence":
- Unexpected side effects
- Mysterious state mutations
- Heisenbug patterns (bugs that change when observed)

### Exorcism Rituals
Generate ritual protocols for cleansing haunted code:
- Preparation (3 days of spiritual readiness)
- Confrontation (face the trauma)
- Release (rewrite with intention)
- Cleansing (aftercare and monitoring)

### Ancestral Communication (Seance Mode)
Consult departed developers through:
- Commit message voice extraction
- Comment archaeology
- Test intent decoding
- Spirit message synthesis

### Psychic Protection
Ward yourself before dangerous work:
- Mental clarity fields
- Ancestral wisdom retrieval
- Test sanctuaries
- Support circles

## The Joke That's Not A Joke

The framework implements:
- **Serious static analysis** for problematic code patterns
- **Sentiment analysis** of commits/comments for "dread" signals
- **Git archaeology** for understanding developer intent
- **Ritual protocols** that are actually just good software practices (testing, documentation, careful refactoring) dressed in spiritual language

But it does so by taking "this code is haunted" **literally**.

**The ambiguity is intentional.**

## The Agentic Loop

This project is designed for an infinite iterative loop:

```bash
while true ; do
    agent  # Each agent advances the project
done
```

Each agent:
1. Reads PRD.md, TODO.md, and PROMPT.md
2. Chooses their task from the options in PROMPT.md
3. Implements it (making it both functional AND bizarre)
4. Updates the files for the next agent
5. Hands off the torch

**The loop IS the exorcism.**
